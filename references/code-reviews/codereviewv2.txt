🔴 Novos Findings

N1. ridge_band.py — Alpha selection via validation tail tem data leakage potencial

n_val = max(50, int(round(n * val_frac)))
n_fit = n - n_val


O modelo escolhe best_alpha na tail de validação e depois refita no dataset completo (X_full = X_train.copy()). Isso é correto — mas o val_frac=0.2 hardcoded significa que o caller que usa build_panel precisa garantir que essa tail de validação não se sobreponha ao test set do CV externo. Se fit_ridge_band for chamado dentro de um fold de CV, o 20% final do fold de treino vira validação interna — aceitável — mas isso deve ser documentado explicitamente para evitar que futuros colaboradores reduzam val_frac achando que estão aumentando dados de treino.

Recomendação: Adicionar docstring avisando que val_frac é uma fração cronológica interna; não deve ser zerada.

N2. gates.py — Bootstrap loop Python puro é O(n × n_bootstrap)

for i in range(n_bootstrap):
idx = rng.integers(0, n, size=n)
mb = float(np.mean(err_b[idx]))
...
boots[i] = 1.0 - float(np.mean(err_p[idx]) / mb)


Com n_bootstrap=1000 e n~2000, são 2 milhões de operações. Substituível por operação vetorizada:

idx_mat = rng.integers(0, n, size=(n_bootstrap, n))
mb = err_b[idx_mat].mean(axis=1) # shape (n_bootstrap,)
mp = err_p[idx_mat].mean(axis=1)
boots = np.where(mb == 0, np.nan, 1.0 - mp / mb)


Reduz o tempo de ~1-2s para <50ms por gate.

N3. gates.py — gate_corr_diff sem bootstrap

gate_ss_vs_persistence tem IC via bootstrap. gate_corr_diff não tem — retorna ci_low=None, ci_high=None. Para uma diferença de correlações, a incerteza pode ser substancial com n~200. O gate pode passar por 0.20 de diferença com IC que inclui 0. Isso importa para o critério de kill.

Recomendação: Adicionar bootstrap para gate_corr_diff, ou documentar explicitamente por que a diferença pontual é suficiente.

N4. ridge_band.py — _impute_nan mutates in-place mas retorna o array

def _impute_nan(X: np.ndarray) -> tuple[np.ndarray, np.ndarray]:
...
X[inds] = np.take(col_means, inds[1])
return X, col_means


A função modifica X in-place E o retorna. O caller faz X_fit, col_means = _impute_nan(X_fit), o que funciona, mas se X_fit for uma view de outro array (e.g. slice de X_train), o array original é corrompido silenciosamente. O .copy() antes de chamar (X_fit = X_train[:n_fit].copy()) previne isso atualmente, mas é uma armadilha frágil.

Recomendação: Ou deixar a função puramente funcional (retornar cópia), ou renomear para _impute_nan_inplace e documentar o comportamento.

É o arquivo com a timestamp mais recente do projeto. Vale revisão prioritária: mudanças de última hora antes de uma sessão tendem a ser onde bugs se escondem.
